Research methodology

XGBoost, CNN-LSTM, autoencoder, and ensemble workflow

The exported notebook trains several models, stores reproducible preprocessing objects, and saves reports that this app reads directly.

01

Preprocessing

Original attack labels are mapped into five classes. The attack and difficulty columns are removed from the feature matrix, while protocol type, service, and flag are one-hot encoded.

02

XGBoost foundation

A multi-class softprob XGBoost model is trained with stratified 5-fold cross-validation. The final model is also trained on the full training split.

03

Stacked deep input

Top gain-ranked XGBoost features are concatenated with model probability outputs, scaled again, and reshaped for the CNN-LSTM stage.

04

CNN-LSTM classifier

The neural model uses convolutional layers, recurrent sequence modelling, regularization, and focal loss to address imbalanced attack categories.

05

Autoencoder anomaly model

The autoencoder learns reconstruction patterns and applies a threshold for binary anomaly decisions, producing separate summary metrics.

06

Free static deployment

The hosted tester embeds only the final XGBoost model and preprocessing metadata, avoiding paid APIs, hosted databases, or paid inference services.

Deployment stance

No paid service dependency

Netlify

Configured with a zero-build `netlify.toml` that publishes the `webapp` directory.

GitHub Pages

The `webapp` folder is static and includes `.nojekyll`, so it can be hosted from a repository branch or docs-style folder.

Cloudflare Pages

No framework adapter is required because the app has no server-side runtime.

Hugging Face Spaces

The same static app can be placed in a Static Space without allocating paid compute.
